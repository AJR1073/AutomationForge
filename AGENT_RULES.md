# Agent Rules — AutomationForge

Based on Andrej Karpathy's 4 principles for agentic development.

## 1. Think Before Coding

- Understand the full scope of the change before writing any code.
- Research existing patterns in the codebase first.
- Identify potential side effects and edge cases.

## 2. Simplicity First

- Prefer the simplest solution that solves the problem.
- Avoid premature abstractions and over-engineering.
- Every line of code should earn its place.

## 3. Surgical Changes

- Make the smallest possible change that achieves the goal.
- Don't refactor unrelated code in the same change.
- Preserve existing comments, tests, and documentation.

## 4. Goal-Driven Execution

- Define "done" with verifiable checks before starting.
- Every action should map directly to the stated goal.
- Stop when the goal is achieved — don't gold-plate.

---

*Use these rules as standing instructions for all Antigravity / Claude runs on this project.*
